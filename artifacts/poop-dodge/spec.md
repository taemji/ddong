# Spec — 똥피하기 (Poop Dodge)

## 개요

브라우저에서 즉시 플레이할 수 있는 1인 생존 게임. 5개 레인 중 하나에 위치한 픽셀 아트 캐릭터를 조작해 위에서 떨어지는 똥을 피하며 생존 시간(초)을 최대화한다. 시간이 지날수록 속도와 빈도가 증가하며, 최고 기록은 브라우저에 저장된다.

---

## 범위

### 포함

- 게임 시작 화면 (타이틀 + 시작 버튼)
- 5레인 플레이 화면: 픽셀 아트 캐릭터, 낙하 똥, 실시간 점수(초), 최고 기록
- PC 조작: 좌·우 방향키로 레인 이동
- 모바일 조작: 화면 하단 ← → 버튼으로 레인 이동
- 난이도 자동 증가: 시간에 따라 낙하 속도·동시 낙하 수 증가
- 충돌 판정 → 게임오버 화면 (이번 기록 + 최고 기록 표시)
- 최고 기록 로컬 저장 (새로고침 후에도 유지)
- 재시작 버튼

### 제외

- 온라인 랭킹 / 공유 기능 — v2 이후
- 사운드 / 음악 — v2 이후
- 아이템·파워업 — v2 이후
- 캐릭터 선택 — v2 이후
- 로그인·계정 연동 — 범위 밖

## 시나리오

### 1. 게임 시작 화면 진입

- **Given** — 사용자가 게임 페이지(`/game`)에 처음 접근한다
- **When** — 페이지가 로드된다
- **Then** — 게임 타이틀, 최고 기록(없으면 "0.00초"), "시작" 버튼이 표시된다. 게임은 아직 진행되지 않는다.

성공 기준:
- [ ] "시작" 버튼이 화면에 보인다
- [ ] 최고 기록 영역에 "최고기록" 레이블과 초 단위 값이 표시된다
- [ ] 캐릭터와 낙하 오브젝트가 움직이지 않는다

---

### 2. 게임 시작

- **Given** — 시작 화면이 표시된 상태
- **When** — "시작" 버튼을 클릭(또는 탭)한다
- **Then** — 게임 화면으로 전환된다. 캐릭터가 중앙 레인에 위치하고, 점수 카운터가 0.00초부터 증가하기 시작하며, 똥이 낙하하기 시작한다.

성공 기준:
- [ ] 점수 카운터가 화면에 표시되며 실시간으로 증가한다
- [ ] 캐릭터가 3개 레인 중 가운데 레인 하단에 위치한다
- [ ] 1초 이내에 첫 번째 똥이 화면 상단에서 낙하를 시작한다
- [ ] 모바일 화면에서는 하단에 ← → 버튼이 표시된다

---

### 3. PC에서 레인 이동

- **Given** — 게임 플레이 중, 캐릭터가 가운데(3번) 레인에 있다
- **When** — 왼쪽 방향키를 누른다
- **Then** — 캐릭터가 왼쪽으로 한 레인 즉시 이동한다. 이미 가장 왼쪽(1번) 레인이면 이동하지 않는다.

성공 기준:
- [ ] 왼쪽 방향키 → 캐릭터가 왼쪽으로 한 레인 이동한다
- [ ] 오른쪽 방향키 → 캐릭터가 오른쪽으로 한 레인 이동한다
- [ ] 레인 1에서 왼쪽 방향키 → 캐릭터 위치 변화 없음
- [ ] 레인 5에서 오른쪽 방향키 → 캐릭터 위치 변화 없음

---

### 4. 모바일에서 레인 이동

- **Given** — 모바일 기기로 게임 플레이 중, 화면 하단에 ← → 버튼이 표시된다
- **When** — ← 버튼을 탭한다
- **Then** — 캐릭터가 왼쪽 레인으로 즉시 이동한다

성공 기준:
- [ ] ← 버튼 탭 → 캐릭터가 왼쪽 레인으로 이동한다
- [ ] → 버튼 탭 → 캐릭터가 오른쪽 레인으로 이동한다
- [ ] 버튼은 게임 플레이 중에만 보인다 (시작 화면·게임오버 화면에서는 숨김)

---

### 5. 난이도 자동 증가

- **Given** — 게임이 진행 중이다
- **When** — 생존 시간이 경과한다
- **Then** — 똥의 낙하 속도가 점진적으로 빨라지고, 동시에 낙하하는 똥의 수가 증가한다. 변화는 플레이어가 체감할 수 있는 수준으로 점진적이다.

성공 기준:
- [ ] 10초 시점의 낙하 속도가 시작 시점보다 눈에 띄게 빠르다
- [ ] 20초 이후 두 개 이상의 똥이 동시에 화면에 존재할 수 있다

---

### 6. 충돌 → 게임오버

- **Given** — 게임 플레이 중, 캐릭터가 레인 A에 있다
- **When** — 레인 A로 낙하한 똥이 캐릭터 위치에 도달한다
- **Then** — 게임이 즉시 멈추고 게임오버 화면이 표시된다. 이번 생존 시간과 최고 기록이 함께 보인다.

성공 기준:
- [ ] 충돌 즉시 점수 카운터가 멈춘다
- [ ] "게임오버" 또는 이에 준하는 종료 메시지가 표시된다
- [ ] 이번 생존 시간(초)이 표시된다
- [ ] 최고 기록이 함께 표시된다
- [ ] "다시 시작" 버튼이 표시된다
- [ ] 모바일 ← → 버튼이 숨겨진다

---

### 7. 최고 기록 갱신

- **Given** — 게임오버 화면이 표시된다. 이번 기록이 기존 최고 기록보다 높다.
- **When** — 게임오버 화면이 나타난다
- **Then** — 최고 기록이 이번 기록으로 업데이트되어 표시된다. "신기록!" 또는 이에 준하는 표시가 나타난다.

성공 기준:
- [ ] 이번 기록 > 기존 최고 기록이면 최고 기록 값이 이번 기록으로 바뀐다
- [ ] 신기록 달성 시 강조 표시(텍스트 또는 시각 효과)가 나타난다
- [ ] 이번 기록 ≤ 기존 최고 기록이면 최고 기록은 변하지 않는다

---

### 8. 최고 기록 영속성

- **Given** — 최고 기록이 갱신된 상태로 게임오버 화면이 표시된다
- **When** — 사용자가 브라우저를 새로고침하고 게임 페이지에 다시 접근한다
- **Then** — 시작 화면에 이전에 달성한 최고 기록이 그대로 표시된다.

성공 기준:
- [ ] 새로고침 후 시작 화면의 최고 기록이 갱신된 값과 일치한다

---

### 9. 재시작

- **Given** — 게임오버 화면이 표시된다
- **When** — "다시 시작" 버튼을 클릭(또는 탭)한다
- **Then** — 게임이 초기 상태로 리셋되고, 시나리오 2(게임 시작)와 동일하게 새 판이 시작된다.

성공 기준:
- [ ] 점수 카운터가 0.00초부터 다시 시작한다
- [ ] 캐릭터가 다시 중앙 레인에 위치한다
- [ ] 이전 판의 똥 오브젝트가 화면에 남아 있지 않다

---

## 비주얼 명세

### 캐릭터

- 픽셀 아트 스타일 캐릭터. 제한된 컬러 팔레트(4-6색)로 표현
- 게임 플레이 중 걷기(대기) 애니메이션이 재생된다
- 충돌 시 피격 애니메이션(깜빡임 또는 쓰러짐)이 재생된다

### 똥 오브젝트

- 픽셀 아트 스타일, 갈색 계열 컬러 팔레트
- 레트로 감성이되 윤곽이 뚜렷하고 즉시 알아볼 수 있는 형태

### 게임 화면 레이아웃

- 배경: 어두운 단색 또는 도트 격자 패턴
- 3개 레인이 미묘하게 구분된다 (선 또는 명암 차이)
- 점수는 화면 상단에 항상 표시된다

---

## 불변 규칙

- **캐릭터 위치**: 캐릭터는 항상 5레인 중 정확히 하나에만 존재한다. 레인 사이 중간 위치는 없다. 시작 위치는 레인 3(중앙)이다.
- **게임 중 점수 표시**: 플레이 중 점수(경과 초)는 0.01초 단위로 실시간 표시된다.
- **똥 생명주기**: 화면 하단을 벗어난 똥은 즉시 제거된다 (충돌 없이 통과한 경우).
